# 01 - 项目概述

## 1. 项目简介

**个人健康管理系统**是一个前后端分离的单体 Web 应用，以健康档案的增删改查（CRUD）为核心业务，集成 Spring AI 实现单条健康记录的智能化分析建议。项目结构清晰、无微服务与中间件依赖，适合作为 Java Web 课程设计或毕业设计。

## 2. 设计目标

| 目标 | 说明 |
|------|------|
| 业务为主 | 约 95% 代码为传统 CRUD 业务 |
| AI 轻量 | 仅 1 个 AI 接口，无对话历史、无流式、无登录 |
| 单体架构 | 无 Nacos、无 Redis、无 MQ |
| 前后端分离 | Vue3 前端 + Spring Boot 后端，RESTful 通信 |
| 规范统一 | 统一响应、全局异常、参数校验、接口文档 |

## 3. 功能清单

### 3.1 健康档案管理（CRUD）

- 分页表格展示健康数据
- 多条件筛选（时间范围、BMI 区间等）
- 新增 / 编辑记录（表单校验）
- 单条逻辑删除、批量逻辑删除
- 根据 ID 查询详情
- BMI 根据身高体重自动计算

### 3.2 健康记录字段

| 字段 | 说明 |
|------|------|
| 身高 | cm |
| 体重 | kg |
| BMI | 自动计算 |
| 心率 | 次/分 |
| 血压 | 如 120/80 |
| 运动时长 | 分钟 |
| 睡眠时间 | 小时 |
| 饮食习惯 | 文字描述 |
| 记录时间 | 数据采集时间 |

### 3.3 AI 健康分析

- 表格每行提供【生成健康建议】按钮
- 传入健康记录 ID，AI 返回三类建议：
  - 健康改善建议
  - 运动建议
  - 饮食建议

## 6. 项目亮点（答辩可用）

1. **标准分层架构**：Controller → Service → Mapper → Entity，职责清晰
2. **统一响应封装**：`Result<T>` + `ResultCode` 枚举
3. **全局异常处理**：业务异常、参数校验、系统异常分层捕获
4. **MyBatis-Plus 增强**：分页插件、逻辑删除、自动填充创建/更新时间
5. **前后端分离联调**：Vite 开发代理 + 后端 CORS 跨域
6. **AI 能力集成**：Spring AI 对接 OpenAI 兼容 API，极简单接口设计
