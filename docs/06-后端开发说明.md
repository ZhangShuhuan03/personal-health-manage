# 06 - 后端开发说明

## 1. 项目信息

| 项目 | 值 |
|------|-----|
| 目录 | `health-server/` |
| 框架 | Spring Boot 3.2.5 |

## 3. 分层架构

```
Controller 层    接收请求、参数校验、调用 Service
    ↓
Service 层       业务逻辑、事务管理
    ↓
Mapper 层        数据库访问（MyBatis-Plus）
    ↓
Entity 层        数据库表映射
```

## 4. 核心模块

### 4.1 统一响应（common）

```java
// 成功
Result.success(data);

// 失败
Result.fail("错误信息");
Result.fail(ResultCode.BAD_REQUEST);
```

### 4.2 全局异常处理（exception）

| 异常类型 | 处理方式 |
|----------|----------|
| BusinessException | 返回业务错误码和消息 |
| MethodArgumentNotValidException | 返回 400 + 校验信息 |
| NoResourceFoundException | 返回 404 |
| Exception | 返回 500，记录日志 |

### 4.3 MyBatis-Plus 配置

**分页插件：**

```java
interceptor.addInnerInterceptor(new PaginationInnerInterceptor(DbType.MYSQL));
```

**逻辑删除：**

```java
@TableLogic
private Integer deleted;
```

**自动填充：**

```java
@TableField(fill = FieldFill.INSERT)
private LocalDateTime createTime;

@TableField(fill = FieldFill.INSERT_UPDATE)
private LocalDateTime updateTime;
```

### 4.4 跨域配置（CorsConfig）

```java
config.addAllowedOriginPattern("*");
config.addAllowedHeader("*");
config.addAllowedMethod("*");
```

### 4.5 健康记录 Service

`HealthRecordServiceImpl` 核心方法：

| 方法 | 说明 |
|------|------|
| pageQuery | 分页 + 多条件 Lambda 查询 |
| getDetailById | 根据 ID 查详情 |
| add | 新增，自动计算 BMI |
| update | 修改，自动计算 BMI |
| deleteById | 逻辑删除 |
| deleteBatch | 批量逻辑删除 |

条件查询使用 `LambdaQueryWrapper`，支持范围查询和模糊匹配。

## 5. 配置文件说明

`application.yml` 主要配置项：

```yaml
server:
  port: 8080                          # 服务端口

spring:
  datasource:                         # 数据库连接
    url: jdbc:mysql://localhost:3306/health_manage
    username: root
    password: xxx

  ai:
    openai:                           # Spring AI 配置
      api-key: xxx
      base-url: https://xxx/compatible-mode  # 不要以 /v1 结尾
      chat:
        options:
          model: qwen-turbo
          temperature: 0.3

mybatis-plus:
  global-config:
    db-config:
      logic-delete-field: deleted     # 逻辑删除字段
      logic-delete-value: 1
      logic-not-delete-value: 0

knife4j:
  enable: true                        # 开启接口文档
```

## 7. 对象流转

```
前端 JSON
    ↓
HealthRecordDTO（请求校验）
    ↓
HealthRecord（Entity，持久化）
    ↓
HealthRecordVO（响应展示）
    ↓
Result<HealthRecordVO>（统一包装）
```

转换工具：`HealthRecordConverter`（BeanUtils.copyProperties）
