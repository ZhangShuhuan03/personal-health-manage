# 07 - AI 功能说明

## 1. 功能概述

本系统仅集成 **1 个 AI 接口**，实现方式极简：

- 传入健康记录 ID
- AI 根据该条记录的健康数据生成三类建议
- 无登录、无对话历史、无流式输出

## 2. 技术方案

| 项目 | 说明 |
|------|------|
| 框架 | Spring AI 1.0.0-M1 |
| 依赖 | `spring-ai-openai-spring-boot-starter` |
| 模型接入 | OpenAI 兼容 API（支持阿里云百炼等） |
| 核心类 | `AiAnalysisServiceImpl`（约 100 行） |

## 3. 调用流程

```
用户点击【生成健康建议】
    ↓
前端 POST /api/health/ai/suggest { recordId: 1 }
    ↓
HealthRecordController.aiSuggest()
    ↓
AiAnalysisServiceImpl.analyze(recordId)
    ├── 1. 查询健康记录数据
    ├── 2. 拼接 Prompt 模板
    ├── 3. 调用 ChatModel（Spring AI）
    ├── 4. 解析 AI 返回的 JSON
    └── 5. 返回 AiAnalysisVO
    ↓
前端弹窗展示三类建议
```

## 4. Prompt 模板

```
你是一位专业的健康顾问。请根据以下健康记录数据，分别给出简洁实用的建议（每项100字以内）：

身高: {height} cm
体重: {weight} kg
BMI: {bmi}
心率: {heartRate} 次/分
血压: {bloodPressure} mmHg
运动时长: {exerciseDuration} 分钟
睡眠时间: {sleepTime} 小时
饮食习惯: {dietHabit}

请严格按以下 JSON 格式返回，不要包含其他内容：
{"healthAdvice":"健康改善建议","exerciseAdvice":"运动建议","dietAdvice":"饮食建议"}
```

## 5. 配置说明

### 5.1 application.yml

```yaml
spring:
  ai:
    openai:
      api-key: your-api-key
      base-url: https://your-host/compatible-mode   # 不要以 /v1 结尾
      chat:
        options:
          model: qwen-turbo
          temperature: 0.3
```

### 5.2 配置要点

| 配置项 | 说明 |
|--------|------|
| api-key | 服务商提供的 API 密钥 |
| base-url | OpenAI 兼容接口地址，**不要**以 `/v1` 结尾 |
| model | 模型名称，需与服务商匹配 |
| temperature | 0.3，较低值使输出更稳定 |

### 5.3 为什么 base-url 不要以 /v1 结尾？

Spring AI 会自动拼接 `/v1/chat/completions`，若 base-url 已含 `/v1`，实际请求路径会变成：

```
❌ https://host/compatible-mode/v1/v1/chat/completions
✅ https://host/compatible-mode/v1/chat/completions
```

## 6. 核心代码

### 6.1 Service 接口

```java
public interface AiAnalysisService {
    AiAnalysisVO analyze(Long recordId);
}
```

### 6.2 调用 AI

```java
ChatResponse chatResponse = chatModel.call(new Prompt(new UserMessage(prompt)));
String response = chatResponse.getResult().getOutput().getContent();
```

### 6.3 响应解析

AI 返回 JSON 格式，后端解析为 `AiAnalysisVO`：

```java
public class AiAnalysisVO {
    private String healthAdvice;     // 健康改善建议
    private String exerciseAdvice;   // 运动建议
    private String dietAdvice;       // 饮食建议
}
```

若 AI 返回格式不标准（如包含 markdown 代码块），会自动清洗后解析；解析失败则返回原始文本。

## 7. 接口说明

```
POST /api/health/ai/suggest
```

**请求：**

```json
{ "recordId": 1 }
```

**响应：**

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "healthAdvice": "您的BMI处于正常范围...",
    "exerciseAdvice": "建议每周进行3-5次有氧运动...",
    "dietAdvice": "保持三餐规律，增加蔬菜水果摄入..."
  }
}
```

## 8. 错误处理

| 场景 | 处理方式 |
|------|----------|
| 记录不存在 | 抛出 BusinessException |
| AI 返回为空 | 抛出 BusinessException |
| API 调用失败 | 捕获异常，返回友好提示 |
| JSON 解析失败 | 降级返回原始 AI 文本 |
