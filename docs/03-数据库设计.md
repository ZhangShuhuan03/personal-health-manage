# 03 - 数据库设计

## 1. 数据库信息

| 项目 | 值 |
|------|-----|
| 数据库名 | `health_manage` |
| 字符集 | `utf8mb4` |
| 排序规则 | `utf8mb4_unicode_ci` |
| 引擎 | InnoDB |

## 2. 表结构

### health_record（健康记录表）

| 字段 | 类型 | 允许空 | 默认值 | 说明 |
|------|------|--------|--------|------|
| id | BIGINT | NO | AUTO_INCREMENT | 主键 ID |
| height | DECIMAL(5,2) | YES | NULL | 身高（cm） |
| weight | DECIMAL(5,2) | YES | NULL | 体重（kg） |
| bmi | DECIMAL(4,2) | YES | NULL | BMI 指数 |
| heart_rate | INT | YES | NULL | 心率（次/分） |
| blood_pressure | VARCHAR(20) | YES | NULL | 血压（如 120/80） |
| exercise_duration | INT | YES | NULL | 运动时长（分钟） |
| sleep_time | DECIMAL(4,1) | YES | NULL | 睡眠时间（小时） |
| diet_habit | VARCHAR(500) | YES | NULL | 饮食习惯 |
| record_time | DATETIME | NO | - | 记录时间 |
| deleted | TINYINT | NO | 0 | 逻辑删除：0-未删除 1-已删除 |
| create_time | DATETIME | NO | CURRENT_TIMESTAMP | 创建时间 |
| update_time | DATETIME | NO | CURRENT_TIMESTAMP | 更新时间（自动更新） |

### 索引

| 索引名 | 字段 | 说明 |
|--------|------|------|
| PRIMARY | id | 主键 |
| idx_record_time | record_time | 按记录时间查询 |
| idx_deleted | deleted | 逻辑删除过滤 |

## 3. ER 关系

本项目仅一张业务表，无表间关联：

```
health_record
├── 基础健康指标（身高、体重、BMI、心率、血压）
├── 生活习惯（运动时长、睡眠、饮食）
├── 时间字段（record_time、create_time、update_time）
└── 逻辑删除（deleted）
```

## 5. 逻辑删除说明

通过 MyBatis-Plus 全局配置实现：

```yaml
mybatis-plus:
  global-config:
    db-config:
      logic-delete-field: deleted
      logic-delete-value: 1
      logic-not-delete-value: 0
```

实体类使用 `@TableLogic` 注解，调用 `removeById()` / `removeByIds()` 时自动将 `deleted` 置为 1，查询时自动过滤已删除记录。

## 6. BMI 计算规则

新增/修改时，若提供了身高和体重，后端自动计算 BMI：

```
BMI = 体重(kg) / [身高(m)]²
```

示例：身高 175cm、体重 70kg → BMI = 70 / (1.75 × 1.75) = 22.86
