# 02 - 环境搭建与运行

## 1. 环境要求

| 工具 | 版本要求 | 说明 |
|------|----------|------|
| JDK | 17+ | 后端运行 |
| Maven | 3.8+ | 后端构建 |
| MySQL | 8.x | 数据库 |
| Node.js | 16+（推荐 18+） | 前端运行 |
| npm | 8+ | 前端包管理 |

## 2. 数据库初始化

### 方式一：命令行

```bash
mysql -u root -p < health-server/sql/init.sql
```

### 方式二：客户端执行

1. 打开 Navicat / DataGrip / MySQL Workbench
2. 执行 `health-server/sql/init.sql` 全部内容
3. 确认数据库 `health_manage` 及表 `health_record` 已创建

### 修改数据库连接

编辑 `health-server/src/main/resources/application.yml`：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/health_manage?useUnicode=true&characterEncoding=utf-8&serverTimezone=Asia/Shanghai&useSSL=false&allowPublicKeyRetrieval=true
    username: root      # 改为你的用户名
    password: your_password  # 改为你的密码
```

## 3. AI 配置（可选）

AI 功能需配置 OpenAI 兼容 API，编辑 `application.yml`：

```yaml
spring:
  ai:
    openai:
      api-key: your-api-key-here
      # 不要以 /v1 结尾，Spring AI 会自动拼接
      base-url: https://your-api-host/compatible-mode
      chat:
        options:
          model: qwen-turbo    # 按服务商调整
          temperature: 0.3
```

> 未配置 API Key 时，CRUD 功能正常，仅 AI 分析接口不可用。

### 阿里云百炼示例

```yaml
spring:
  ai:
    openai:
      api-key: sk-xxx
      base-url: https://dashscope.aliyuncs.com/compatible-mode
      chat:
        options:
          model: qwen-turbo
```

## 4. 启动后端

```bash
cd health-server
mvn spring-boot:run
```

或使用 IDE（IntelliJ IDEA）直接运行 `HealthManageApplication.java`。

启动成功后：

- 后端地址：http://localhost:8080
- 接口文档：http://localhost:8080/doc.html

### 生产打包

```bash
cd health-server
mvn clean package -DskipTests
java -jar target/personal-health-manage-1.0.0.jar
```

## 5. 启动前端

```bash
cd health-web
npm install
npm run dev
```

启动成功后访问：http://localhost:5173

### 生产构建

```bash
cd health-web
npm run build
npm run preview
```

构建产物在 `health-web/dist/` 目录，可部署到 Nginx 等静态服务器。

## 6. 联调说明

### 开发环境请求流程

```
浏览器 → Vite Dev Server (:5173)
         ├── /health          → 前端页面路由
         └── /api/health/*    → 代理到后端 (:8080)
```

### 注意事项

1. **先启后端，再启前端**
2. 前端开发模式下 API 通过 Vite 代理转发，无需关心跨域
3. 生产环境需将 `health-web/src/utils/request.js` 中 `baseURL` 指向实际后端地址
4. 后端 CORS 已配置，允许 `http://localhost:*` 跨域访问

## 7. 常见问题

### Q: Maven 下载 Spring AI 依赖失败？

项目使用 Spring Milestone 仓库，`pom.xml` 已配置：

```xml
<repository>
    <id>spring-milestones</id>
    <url>https://repo.spring.io/milestone</url>
</repository>
```

### Q: 前端 `crypto.getRandomValues` 报错？

Node.js 版本过低，需 16+。项目已降级 Vite 至 4.x 兼容 Node 16。

### Q: AI 接口返回 500？

1. 检查 `api-key` 是否正确
2. 检查 `base-url` 不要以 `/v1` 结尾
3. 检查 `model` 名称是否与服务商匹配

### Q: 页面刷新后接口 404？

确认 API 路径使用 `/api/health/*` 前缀，与前端页面路由 `/health` 分离。
