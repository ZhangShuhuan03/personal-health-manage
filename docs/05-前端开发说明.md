## 3. 核心模块说明

### 3.1 Axios 封装（`utils/request.js`）

```javascript
const request = axios.create({
  baseURL: import.meta.env.DEV ? '' : 'http://localhost:8080',
  timeout: 30000
})
```

- **开发模式**：`baseURL` 为空，请求走 Vite 代理
- **生产模式**：直连后端 `http://localhost:8080`
- **响应拦截**：`code === 200` 返回 data，否则 `ElMessage.error` 提示

### 3.2 API 层（`api/health.js`）

| 函数 | 方法 | 路径 |
|------|------|------|
| getHealthList | GET | /api/health/list |
| getHealthById | GET | /api/health/{id} |
| addHealth | POST | /api/health/add |
| updateHealth | PUT | /api/health/update |
| deleteHealth | DELETE | /api/health/{id} |
| batchDeleteHealth | DELETE | /api/health/batch |
| aiSuggest | POST | /api/health/ai/suggest |

### 3.3 路由（`router/index.js`）

| 路径 | 组件 | 说明 |
|------|------|------|
| `/` | MainLayout | 重定向到 /health |
| `/health` | HealthRecord.vue | 健康记录管理 |

> 页面路由 `/health` 与 API 路径 `/api/health/*` 分离，避免代理冲突。

### 3.4 Vite 代理（`vite.config.js`）

```javascript
proxy: {
  '/api': {
    target: 'http://localhost:8080',
    changeOrigin: true
  }
}
```

仅 `/api` 前缀的请求代理到后端，前端页面路由不受影响。

## 4. 页面功能

### 4.1 布局（MainLayout.vue）

- **左侧**：深色侧边栏，菜单项「健康记录管理」
- **顶部**：系统标题「个人健康管理系统」
- **内容区**：`<router-view />` 渲染子页面

### 4.2 健康记录页（HealthRecord.vue）

#### 搜索区域

- 记录时间范围（日期时间选择器）
- BMI 区间（最小值 ~ 最大值）
- 查询 / 重置按钮

#### 数据表格

| 列 | 说明 |
|----|------|
| 选择框 | 批量删除用 |
| ID | 主键 |
| 身高/体重/BMI | 数值展示，BMI 带颜色标签 |
| 心率/血压 | 健康指标 |
| 运动/睡眠/饮食 | 生活习惯 |
| 记录时间 | 数据采集时间 |
| 操作 | 编辑 / 生成健康建议 / 删除 |

#### BMI 颜色标签

| BMI 范围 | 颜色 | 含义 |
|----------|------|------|
| < 18.5 | 橙色 | 偏瘦 |
| 18.5 ~ 24 | 绿色 | 正常 |
| 24 ~ 28 | 橙色 | 偏胖 |
| > 28 | 红色 | 肥胖 |

#### 弹窗

1. **新增/编辑弹窗**：8 项表单 + 必填校验
2. **AI 建议弹窗**：展示健康改善、运动、饮食三类建议

## 7. 生产部署建议

1. 执行 `npm run build` 生成 `dist/` 目录
2. 将 `dist/` 部署到 Nginx
3. 配置 Nginx 反向代理 `/api` 到后端

```nginx
server {
    listen 80;
    root /path/to/dist;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://localhost:8080;
    }
}
```

4. 修改 `request.js` 中生产环境 `baseURL` 为实际后端地址
